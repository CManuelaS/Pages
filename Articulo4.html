Cómo Utilizar Git: Un Tutorial Detallado

Git es una de las herramientas más importantes para los desarrolladores que buscan gestionar el control de versiones en sus proyectos. En este tutorial, te guiaré paso a paso sobre cómo usar Git, desde la inicialización de un repositorio hasta la gestión de ramas y la conexión con un repositorio remoto. Todo estará acompañado de comandos útiles y ejemplos prácticos.

1. Inicialización del Repositorio Localmente

Para comenzar, sigue estos pasos para crear un repositorio local en tu computadora:

  1. Paso 1: Crea una carpeta en tu computadora para tu proyecto. Puedes utilizar un proyecto existente, como el de tu blog.
  2. Paso 2: Navega a la carpeta donde está tu proyecto.

Una vez que estés en la carpeta de tu proyecto, ejecuta el siguiente comando para inicializar Git:

git init

Este comando realiza lo siguiente:

  • Crea un repositorio Git vacío en la carpeta actual.
  • Genera una carpeta oculta llamada .git que Git utilizará para almacenar la información del historial del proyecto, cambios realizados, ramas y configuraciones.

Propósito: El comando git init convierte cualquier carpeta en un repositorio Git, permitiéndote rastrear los cambios en los archivos de tu proyecto.

2. Crear un Repositorio en GitHub

Después de haber inicializado tu repositorio localmente, es importante crear un repositorio remoto en GitHub. Sigue estos pasos:

  • Paso 1: Inicia sesión en tu cuenta de GitHub. Si no tienes una cuenta, debes crearla previamente.
  • Paso 2: Haz clic en el ícono + en la esquina superior derecha de la página y selecciona New repository.
  • Paso 3: Asigna un nombre a tu repositorio, por ejemplo, "blog", y luego haz clic en Create repository.
  • Paso 4: Debe aparecerte algo similar a las imágenes anteriores. Si es así, ya has creado tu repositorio.
  • Paso 5: Ahora ve a la carpeta donde se encuentra tu proyecto. Haz clic con el botón derecho y selecciona "Mostrar más opciones", luego elige "Open Git Bash here" o "Abrir Git Bash aquí".
  • Paso 6: Al seleccionar esta opción, se abrirá la terminal de Git. Asegúrate de tener Git instalado previamente en tu computadora. Si no lo tienes instalado, puedes descargarlo desde el siguiente enlace: Descargar Git Es obligatorio tener Git instalado para subir tu proyecto a GitHub.

3. Agregar Archivos al Repositorio y Hacer un Commit

Ahora que has creado tu repositorio local y remoto, es momento de agregar los archivos a Git y hacer tu primer commit:

  • Paso 7: En la terminal de Git Bash, ejecuta git add . para agregar todos los archivos de tu proyecto.
  • Paso 8: Escribe git commit -m "first commit" para registrar los cambios. Después, escribe git branch -M main para asegurarte de que estás trabajando en la rama principal.

4. Conectar el Repositorio Local a GitHub

  • Paso 9: Ahora es momento de vincular el repositorio local con el remoto. Para ello, utiliza el siguiente comando:
git remote add origin https://github.com/tu_usuario/nombre_repositorio.git
  • Paso 10: Finalmente, ya esta tu proyecto en GitHub con el comando:
git push -u origin main

3. Creación de Branches en Git

Las ramas (branches) en Git permiten trabajar en diferentes versiones del proyecto simultáneamente. Esto es útil para desarrollar nuevas funcionalidades, corregir errores o experimentar con nuevas ideas sin afectar el código principal.

¿Cuándo utilizar branches?

Crear ramas es recomendable cuando necesitas trabajar en una nueva característica o realizar cambios importantes en el proyecto sin afectar la rama principal (generalmente llamada main o master). Esto permite realizar cambios de forma aislada y probar nuevas ideas sin interrumpir el flujo de trabajo principal.

1. Crear una nueva rama

Para crear una nueva rama, utiliza el siguiente comando:

git branch nombre_rama

Este comando crea una rama llamada nombre_rama basada en la rama actual. La nueva rama es una copia del estado actual del proyecto.

2. Cambiar a la nueva rama

Una vez que la rama ha sido creada, debes cambiar a ella para empezar a trabajar en ella. Utiliza el siguiente comando:

git checkout nombre_rama

O bien, puedes usar el comando más reciente:

git switch nombre_rama

3. Trabajar en la nueva rama

Ahora que estás en la nueva rama, puedes realizar cambios en el código, agregar archivos, y hacer commits sin afectar la rama principal.

4. Fusionar ramas con merge

Una vez que hayas terminado de trabajar en la rama y quieras incorporar los cambios a la rama principal, debes usar el comando merge. Primero, cambia a la rama principal: 

git checkout main

Luego, utiliza el siguiente comando para fusionar la rama en la rama principal:

git merge nombre_rama

Esto incorpora los cambios realizados en nombre_rama a la rama main. Si hay conflictos entre los archivos modificados en ambas ramas, Git te pedirá que los resuelvas manualmente.

Ejemplo práctico

Supongamos que estás trabajando en una nueva funcionalidad y has creado una rama llamada feature-login. Después de realizar y probar los cambios, deseas fusionar estos cambios en la rama principal:

  1. Crea y cambia a la rama feature-login:
    2. git branch feature-login
            git checkout feature-login
  2. Haz cambios en el proyecto y realiza commits:
    3. git add .
            git commit -m "Add login feature"
  3. Cambia a la rama principal:
    4. git checkout main
  4. Fusiona la rama feature-login en main:
    5. git merge feature-login
  5. Resuelve cualquier conflicto si es necesario y realiza un commit para completar la fusión.

Con estos pasos, has creado, trabajado y fusionado ramas en Git, permitiendo un flujo de trabajo organizado y eficiente.
